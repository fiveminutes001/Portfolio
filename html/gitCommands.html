Creating a repository

To initiate a repo

git init

To clone a repo

git clone https://github.com/fiveminutes001/Portfolio.git

To update local repo (fetch + merge)

git pull https://github.com/fiveminutes001/Portfolio.git

Updating a repository

To shorten commands to one line

git add . && git commit -m "updated file" && git push

To update remote repo

git add . Or git add -A //with higher directories
git diff --check
git commit -m "initial commit 2" Orgit commit -v (for longer message)
git push origin master

To fix commit message

git commit --amend

To update all modules

git submodule update

To update submodule in main directory

In submodule local dir: git add . && git commit -m "updated file" && git push gitMaster master
In main local dir: git submodule update --remote udacity/JsAndTheDOM
In main local dir: git add . && git commit -m "updated file" && git push
In submodule remote dir: git pull origin master
In main remote dir: git pull

To find out what's the origin

git remote -v

To show remote details

git remote show origin

To add a local source to remote repo

git remote add origin https://github.com/fiveminutes001/Portfolio.git/

You can add any number of remotes with the command git remote add. You can then push to one of them by explicitly stating the remote in git push. E.g. git push foobar master to push the current branch to master on remote foobar

Making life easier

To shorten git commands (pull and push)

$ git remote add progit https://github.com/progit/progit2.git (1)
$ git fetch progit (2)
$ git branch --set-upstream-to=progit/master master (3)
$ git config --local remote.pushDefault origin (4)

Add the source repository and give it a name. Here, I have chosen to call it progit. Get a reference on progit's branches, in particular master. Set your master branch to fetch from the progit remote. Define the default push repository to origin.

http://git-scm.com/book/en/v2/GitHub-Contributing-to-a-Project
